# Решение проблемы с памятью Node.js

## Проблема
Ошибка: `FATAL ERROR: invalid table size Allocation failed - JavaScript heap out of memory`

Это происходит потому, что Node.js по умолчанию имеет ограничение памяти около 1.5-2GB, а React приложение при компиляции требует больше.

## ✅ РЕШЕНИЕ (УЖЕ ПРИМЕНЕНО)

### Способ 1: Просто запустить (РЕКОМЕНДУЕТСЯ) ⭐

**Просто выполните:**
```bash
npm start
```

Настройка памяти уже встроена в `package.json`! Скрипт `start` автоматически использует `node --max-old-space-size=8192`.

**Или используйте готовые скрипты:**
- **PowerShell:** `.\start.ps1`
- **CMD:** `start.bat`

### Способ 2: Если проблема сохраняется

Увеличьте память в `package.json`:
```json
"start": "node --max-old-space-size=12288 node_modules/react-scripts/bin/react-scripts.js start"
```
(12288 = 12GB)

### Способ 3: Создать файл .env.local (альтернатива)

Создайте файл `.env.local` в корне проекта со следующим содержимым:
```
NODE_OPTIONS=--max-old-space-size=8192
```

## Что было сделано

- ✅ **Обновлен `package.json`** - скрипт `start` теперь напрямую использует `node --max-old-space-size=8192`
- ✅ Установлен лимит памяти 8GB (8192 MB)
- ✅ Обновлены скрипты `start.bat` и `start.ps1`
- ✅ Убрано дублирование `ThemeProvider` из `index.tsx`

## Рекомендации

1. **Просто выполните `npm start`** - настройка памяти уже встроена ⭐
2. Если проблема сохраняется, увеличьте значение до 12288 (12GB) в `package.json`
3. Убедитесь, что у вас достаточно свободной оперативной памяти (минимум 8GB рекомендуется)
4. Закройте другие приложения, потребляющие много памяти
